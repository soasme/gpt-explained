# GPT Explained

*A hands-on guide to transformer architecture — from tokens to text generation*

---

This book teaches you how GPT-style language models actually work, from the very first step (turning text into integers) all the way to the final step (turning vectors back into words). It is written in the spirit of *Hands-On Machine Learning with Scikit-Learn and TensorFlow* — every concept is explained intuitively first, then backed by math, then grounded in concrete matrix examples, and finally implemented in code.

**The code language is Lisp.** Not because it is the fashionable choice, but because Lisp is the closest thing to executable math. A function is a formula. A list is a matrix row. The code reads like the equations.

**The diagrams are animated.** Every figure in this book is generated by a Manim script, living alongside the prose. You can reproduce any animation by running the corresponding script in `src/manim/`.

## What you will build

By the end, you will have a complete working microGPT — a minimal but correct GPT implementation in Lisp that you can run and inspect. Not a toy metaphor: actual matrix multiplications, actual softmax, actual multi-head attention.

## Who this is for

You should be comfortable with high-school algebra. You do not need prior ML experience. When a concept like *dot product* or *softmax* appears, a **Math Minute** box explains it on the spot.

## How to read this book

Each chapter follows the same four-beat structure:

1. **The Idea** — what the mechanism does in plain language
2. **The Math** — the equations, explained term by term
3. **The Matrix** — a worked example with small concrete numbers
4. **The Code** — a Lisp implementation, step by step

The chapters build on each other. Read them in order the first time.

# Install dependencies

cd src/manim
for f in ch*.py; do manim -ql "$f"; done
